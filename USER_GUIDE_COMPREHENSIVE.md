## Getting Started

### System Requirements

- **Browser**: Chrome, Firefox, Safari, or Edge (latest versions recommended)
- **Internet**: Stable connection required
- **Screen**: Minimum 1024×768 resolution recommended
- **Cookies**: Must be enabled

### Accessing the Portal

1. Navigate to: `https://affairs.iitg.ac.in` (or your institution's portal URL)
2. You will see the **Login Page** with the IITG Affairs Portal branding
3. Enter your credentials:
   - **User ID**: Your institutional ID (e.g., CS21B001 for students, FAC001 for faculty)
   - **Password**: Your secure password
   - **Role** (optional): Select your role if you have multiple accounts

### First-Time Login

**Important**: If you were imported via CSV bulk upload:
1. Your initial password is temporary and will be provided separately
2. Upon first login, you **must change your password** before accessing the portal
3. Enter a new password that meets the **Password Policy**:
   - 8–16 characters in length
   - At least one uppercase letter (A-Z)
   - At least one lowercase letter (a-z)
   - At least one digit (0-9)
   - At least one special character (!@#$%^&*)
   - Example: `MySecurePass123!`

### Dashboard Overview

After login, you see your **role-based dashboard**:

| Element | Purpose |
|---------|---------|
| **Sidebar Navigation** | Links to your role's features |
| **Top Header** | Profile menu, logout, search |
| **Main Content Area** | Your role-specific dashboard or page |
| **Breadcrumbs** | Shows page hierarchy |

### Course Registration (Enrollment)

**Purpose**: Enroll in course offerings for the current semester

**Step-by-Step Process**:

1. **Navigate** to Student Portal → **Courses** → **Browse & Register**
2. **View Available Courses**: Displays all available course offerings
   - Shows: Course Code, Title, Credits, Faculty, Capacity, Current Enrollment
   - Color-coded: 🟢 Green (seats available) / 🟡 Yellow (filling up) / 🔴 Red (full)

3. **Filter & Search** (Optional):
   - Filter by Department, Credit Range, Faculty
   - Search by Course Code or Title

4. **Select Course**: Click course card to view details:
   - Full description
   - Prerequisites (if any)
   - Schedule (days/times)
   - Faculty contact info
   - **"Enroll Now"** button

5. **Confirm Enrollment**:
   - Click "Enroll Now"
   - System checks capacity in real-time
   - If successful: Course added to "My Enrollments"
   - If full: Message "Course is at capacity. Please try another course."

6. **View My Enrollments**: Shows all enrolled courses:
   - Course details
   - Current grade (if finalized)
   - Drop option (if deadline not passed)

**Important Notes**:
- Enrollment capacity is **strictly enforced** to prevent overbooking
- **Drop deadline**: Specified by the admin; after this date, you cannot drop
- **Prerequisite check**: System verifies you completed prerequisites (if any)
- **Credit limit**: Check with your program advisor for maximum credits per semester

### Assignment Management

**Submitting Assignments**:

1. **View Assignments**: Student Portal → **Assignments** → Select Course
   - Lists all assignments with:
     - Title, Description, Deadline
     - Maximum points, Faculty instructions
     - Allowed file types (PDF, DOCX, etc.)
     - Assignment specifications (downloadable)

2. **Download Assignment Brief**: Click course assignment; view/download faculty specification

3. **Submit Assignment**:
   - Click **"Submit Assignment"** button
   - Drag-and-drop OR browse to select file
   - System validates:
     - File format matches allowed types
     - File size < 10MB
   - If **late**: Shows warning "Submission is late" (but still accepted)
   - Click **"Submit"** to confirm

4. **Confirmation**: Receive confirmation with:
   - Submission time (local timezone)
   - File name and size
   - Late status (if applicable)

5. **Track Submissions**: View all submissions in **My Submissions** tab:
   - Submission status: Pending/Graded
   - Score (once faculty grades)
   - Faculty feedback

**Late Submission Policy**:
- System automatically detects if submitted after deadline
- Late submissions **may incur penalties** (per faculty guidelines)
- Faculty can override deadline if needed

### Requesting Transcripts

**Purpose**: Download official academic transcript (PDF) for admissions, jobs, etc.

**Step-by-Step**:

1. Navigate to: **Documents** → **Transcript Requests**
2. Click **"Request New Transcript"**
3. Fill form:
   - **Number of copies**: e.g., 2
   - **Delivery format**: Digital PDF (instant)
   - **Purpose** (optional): e.g., "Job application"

   - Processing time: Usually < 24 hours

   - Once approved, **"Download"** button appears

### Hostel & Welfare Services

#### Leave Applications

**Purpose**: Apply for hostel leave (exeat) for traveling or going home

**How to Apply**:

1. Navigate to: **Hostel & Welfare** → **Leave Applications**
2. Click **"Apply for Leave"**
3. Fill form:

#### Filing Complaints

**Purpose**: Report hostel maintenance or facility issues

**How to File**:

1. Navigate to: **Hostel & Welfare** → **Complaints**
2. Click **"New Complaint"**
3. Fill form:
   - **Type**: Select from dropdown (Maintenance, Facility, Other)

   - View resolution notes

**Example**: 
- *Type*: Maintenance
- *Title*: "Leaky faucet in bathroom"
- *Description*: "Water leaking from sink faucet, dripping ~50ml/hour"

### Grade Submission

**Purpose**: Submit final grades for all enrolled students

**Submit Grades**:

1. Navigate to: **My Courses** → Select Course → **Grading**
2. View student list with **Grading Table**:
   - Student ID | Name | Grade | GPA Points
   - Empty Grade column (for input)

3. **Enter Grades**:
   - Click on Grade cell
   - Enter letter grade: A, B+, B, C+, C, D, F
   - System converts to GPA points (A=10, B+=9, etc.)
   - Tab to next student or click Submit

4. **Validate**:
   - System checks: All students have grades, grades valid
   - Show error if missing or invalid

5. **Submit All Grades**:
   - Click **"Submit Final Grades"**
   - Confirmation dialog: "This action locks grades. Confirm?"
   - Click **"Confirm"**

6. **After Submission**:
   - Grades **locked** (no changes without admin intervention)
   - System sends: Grades submitted notification to students
   - Grades appear in students' academic records

**Grade Scale**:
```
A  = 10 (90-100%)     | F  = 0  (0-40%)
B+ = 9  (80-89%)      | I  = 0  (Incomplete)
B  = 8  (70-79%)      | W  = -  (Withdrawn)
C+ = 7  (65-69%)      | P  = -  (Pass, non-graded)
C  = 6  (60-64%)      | NG = -  (Not graded)
D  = 5  (50-59%)      |
```

#### Bulk Import Users (CSV)

**Purpose**: Import multiple users at once (students, faculty, etc.)

**Prepare CSV File**:

Format:
```
userId,name,email,role,department
CS21B001,John Doe,john@iitg.ac.in,student,Computer Science
CS21B002,Jane Smith,jane@iitg.ac.in,student,Computer Science
FAC001,Dr. Robert Johnson,robert@iitg.ac.in,faculty,Computer Science
ADM001,Admin User,admin@iitg.ac.in,admin,Administration
```

**Required Columns**:
- `userId`: Unique ID (string)
- `name`: Full name
- `email`: Email address
- `role`: One of [student, faculty, admin, hmc_member]
- `department`: Department/organization

**Steps**:

1. Navigate to: **Admin Portal** → **User Management** → **Bulk Import**
2. Click **"Choose File"** and select your CSV
3. Preview table appears showing:
   - First 5 rows for validation
   - Column mapping (auto-detected)
   - Any parsing errors highlighted

4. **Review Import Summary**:
   - Total records: 150
   - New users to create: 140
   - Users to update: 10
   - Errors: 0

5. Click **"Confirm Import"**
   - System processes file (may take ~10-30 seconds for 1000+ records)
   - Progress bar shows status

6. **Import Results**:
   - Detailed results: Created, Updated, Failed with reasons
   - Export results as CSV for record-keeping
   - Example success: "CS21B001 - Created successfully"
   - Example error: "FAC002 - Email already exists"

**Rollback on Error**: If ANY row fails validation, entire import is rolled back (all-or-nothing transaction)

**Temporary Password**: All new users get temporary password (must change on first login)

## Troubleshooting & FAQ

### Common Issues

**Q: "Invalid Credentials" on Login**
- Check User ID and password capitalization
- Verify caps lock is off
- If first login: Use temp password provided
- Forgot password? Click "Forgot Password" link

**Q: Session Expired; Need to Login Again**
- JWT tokens valid 24 hours
- If logged in multiple days: Re-login needed
- This is normal; helps with security

**Q: Cannot Upload Assignment; "File Type Not Allowed"**
- Check allowed file types (shown in assignment)
- Example: If only PDF/DOC allowed, don't upload .txt
- Rename file to correct extension if needed

**Q: Course Registration Shows "Full" But I See Available Seats**
- System enforces real-time capacity checks
- Another student may enroll simultaneously
- Refresh page to see current status
- Try registering for different course section/time

**Q: Transcript Request Pending > 24 Hours**
- Admin may be reviewing other requests
- Processing time: Usually < 48 hours
- Contact admin if > 3 days

**Q: Cannot Drop Course; Says "Deadline Passed"**
- Drop deadline is set by admin (check calendar)
- After deadline: Cannot drop (course counts toward GPA)
- Contact admin if you believe this is error

**Q: Attendance Shows 0% But I Attended Classes**
- Faculty must mark attendance in system
- Attendance not automatic (based on physical records)
- Verify with your faculty member if discrepancy exists

**Q: Bulk Import Failed; Some Users Created, Some Not**
- Entire import rolled back if any rows fail
- Check error report for reason (duplicate email, etc.)
- Fix CSV and re-import

### FAQ

**Q: What if I forgot my password?**
A: Click "Forgot Password" on login page. Verify your email and user ID. You'll receive a reset link (or token shown in dev mode). Set a new password. Note: Password must meet policy (8-16 chars, upper/lower/digit/special).

**Q: Can I change my role (e.g., from student to faculty)?**
A: No. Your role is set by admin. If you have multiple roles, you can select role at login. Contact admin if role assignment is incorrect.

**Q: How is my attendance tracked?**
A: Faculty marks attendance in class (or uploads CSV). System doesn't track biometric/location. Disputes should be raised with faculty member.

**Q: Can I re-take a course I already took?**
A: Yes. You can re-enroll and new grade will be computed. Old grade may still show in transcript (depending on policy). Check with academic advisor.

**Q: Will my grades be visible to other students?**
A: No. Grades are private. Only you and faculty can see. System enforces this strictly.

**Q: What happens if I don't change my password on first login?**
A: Depends on admin policy. Usually, you can still use temp password, but it's recommended to change immediately for security.

**Q: Can I submit assignment late?**
A: Yes. System accepts late submissions but marks them as "Late" (on submission timestamp). Faculty may apply penalties per course policy.

**Q: How long are my records kept after graduation?**
A: Typically 7-10 years (per institutional policy). Transcripts available indefinitely for download.

**Q: Who do I contact for technical issues?**
A: Email: support@iitg.ac.in or contact your department admin.

## Security & Best Practices

### Password Security

✓ **DO**:
- Use strong passwords (8-16 chars, mixed types)
- Change password every 90 days
- Use unique password (not shared with other services)
- Use password manager to store securely

✗ **DON'T**:
- Share password with friends or roommates
- Write password on paper/post-it
- Use username as part of password
- Reuse passwords across sites

### Account Security

✓ **DO**:
- Log out when done (click Logout button)
- Use secure WiFi (not public/open WiFi for sensitive operations)
- Enable 2FA if available (future feature)
- Report suspicious activity immediately

✗ **DON'T**:
- Leave browser logged in on shared computer
- Click unknown links claiming to be from portal
- Share login credentials
- Access portal on untrusted devices

### Data Privacy

- Your grades are **private** (only you + faculty can see)
- Announcements may be **public** (visible to all enrolled students)
- Transcript requests create **audit trail** (admin can see who requested)
- Messages are **not encrypted** (treat as semi-public)

### Reporting Issues

**Security Concern** (hack, phishing, data leak):
- Email: security@iitg.ac.in
- Include: What happened, when, any screenshots

**Technical Bug**:
- Email: support@iitg.ac.in
- Include: Browser, exact error, steps to reproduce

**Inappropriate Content**:
- Report to admin immediately
- Include: Screenshot, message ID
